## 1. 사업 개요

### 1.1 서비스명

**HR Management System** (인사관리 통합 플랫폼)

### 1.2 비전 (Vision)

중소기업과 개인이 함께 성장하는 차세대 인사관리 생태계를 구축하여, 효율적이고 투명한 고용 관계를 실현한다.

### 1.3 미션 (Mission)

- 법인에게는 효율적인 직원 관리 도구를 제공
- 개인에게는 경력 데이터의 자율적 소유와 관리 권한 보장
- 양측 모두에게 계약 기반의 투명한 데이터 공유 체계 제공
- AI 기술을 활용한 문서 자동화로 인사 업무 생산성 향상

### 1.4 핵심 가치 제안

#### 법인(기업) 고객
- 멀티테넌시 SaaS 기반 독립적 데이터 관리
- 조직도 기반 계층적 인사 관리
- 직원 정보, 급여, 근태, 평가 통합 관리
- 개인-법인 계약 기반 투명한 데이터 공유
- AI 문서 처리로 수기 입력 부담 최소화

#### 개인 사용자
- 자신의 경력 데이터를 직접 소유하고 관리
- 여러 법인과 계약 체결 시 데이터 재사용 가능
- 법인별 공유 범위 설정 가능
- 통합 프로필로 경력 이력 체계적 관리
- 계약 기반 투명한 고용 관계 형성

#### 직원 서브 계정
- 법인이 생성한 계정으로 시작하되, 필요 시 개인 계정으로 전환 가능
- 법인 인사 시스템과 개인 경력 관리의 가교 역할
- 퇴사 후에도 자신의 데이터를 개인 계정으로 보관 가능

## 2. 시장 분석

### 2.1 타겟 시장

#### 주요 타겟 (Primary Target)

**중소기업 (직원 10~200명)**
- 규모: 국내 중소기업 약 700만 개 (2024년 기준)
- 특징: 전용 인사 시스템 도입 비용 부담, 엑셀 또는 수기 관리 의존
- 니즈: 저렴한 비용으로 전문적인 인사 관리 시스템 필요

**스타트업 (직원 5~50명)**
- 규모: 국내 스타트업 약 5만 개 (2024년 기준)
- 특징: 빠른 조직 변화, 유연한 인사 관리 필요
- 니즈: 초기 투자 부담 낮은 SaaS 서비스, 빠른 도입

#### 부차 타겟 (Secondary Target)

**개인 사용자 (구직자, 프리랜서)**
- 규모: 국내 경제활동인구 약 2,900만 명 (2024년 기준)
- 특징: 경력 관리의 중요성 인식 증가, 다양한 고용 형태
- 니즈: 통합 경력 관리, 이력서 자동 생성, 경력증명서 관리

### 2.2 경쟁 환경 분석

#### 기존 인사관리 시스템 (On-Premise)

| 제품 | 강점 | 약점 |
|------|------|------|
| SAP HR | 대기업 검증, 통합성 | 고비용, 복잡한 구축 |
| Oracle HCM | 글로벌 표준, 확장성 | 도입 비용, 유지보수 비용 |

**우리의 차별화**: 중소기업 맞춤형 저비용 SaaS, 빠른 도입

#### 국내 HR SaaS

| 제품 | 강점 | 약점 |
|------|------|------|
| 플렉스 | 국내 근무제도 특화 | 단순 근태 중심 |
| 리멤버 | 명함 관리 통합 | 인사관리 기능 제한적 |
| 잡플래닛 | 기업 정보 제공 | 인사 관리 도구 부재 |

**우리의 차별화**: 개인-법인 양방향 플랫폼, 데이터 자율 소유권, AI 문서 처리

#### 개인 경력 관리 서비스

| 제품 | 강점 | 약점 |
|------|------|------|
| 링크드인 | 글로벌 네트워크 | 국내 경력 관리 부족 |
| 잡코리아/사람인 | 구인구직 통합 | 경력 관리 기능 미흡 |

**우리의 차별화**: 법인 시스템과 연계된 자동 경력 업데이트, 데이터 공유 통제

### 2.3 시장 기회

#### 시장 트렌드

1. **디지털 전환 가속화**
   - 코로나19 이후 중소기업의 클라우드 SaaS 도입 급증
   - 2024년 HR Tech 시장 규모: 약 5조 원 (전년 대비 20% 성장)

2. **유연근무제 확산**
   - 재택근무, hybrid 근무 증가로 디지털 인사 관리 필수화
   - 근태 관리, 평가 시스템의 디지털화 요구 증가

3. **개인정보 보호 강화**
   - 개인정보보호법 강화로 데이터 소유권 의식 증대
   - GDPR 등 글로벌 규제에 부합하는 데이터 관리 필요

4. **AI 기술 활용**
   - OCR, 자연어 처리를 활용한 자동화 수요 증가
   - 생성형 AI를 통한 문서 작성 자동화

#### 시장 진입 기회

1. **중소기업 언택트 솔루션 수요**
   - 대면 인사 관리의 디지털 전환 필요성
   - 저렴한 비용, 빠른 도입 가능한 SaaS 선호

2. **개인 경력 관리 공백**
   - 기존 서비스는 기업 중심 또는 구인구직 중심
   - 개인 주도적 경력 관리 플랫폼 부재

3. **계약 기반 투명성 수요**
   - 프리랜서, 계약직 증가로 계약 기반 관리 중요성 증대
   - 양측 모두 만족하는 투명한 데이터 공유 모델 필요

## 3. 서비스 소개

### 3.1 핵심 기능

#### 3.1.1 법인 계정 기능

**조직 관리**
- 계층적 조직도 구성 (본부 > 사업부 > 부서 > 팀 > 파트)
- 트리 구조 기반 조직 관리
- 조직별 직원 배치 및 이동 관리

**직원 관리**
- 직원 기본정보 관리 (인적사항, 연락처, 주소)
- 학력, 경력, 자격증, 어학, 병역 정보 관리
- 가족 정보, 수상 이력, 프로젝트 이력 관리
- 사진 및 첨부서류 관리 (암호화 저장)

**급여 및 복리후생**
- 급여 정보 관리 (기본급, 수당, 연봉)
- 급여 지급 이력 추적
- 복리후생 (연차, 퇴직금) 관리
- 4대 보험 정보 관리

**계약 관리**
- 개인-법인 간 계약 요청/승인/거절 프로세스
- 계약 유형별 관리 (정규직, 계약직, 인턴, 프리랜서)
- 계약 상태 추적 (대기, 활성, 거절, 종료)
- 데이터 공유 설정 (계약별 세밀한 제어)

**인사평가 및 교육**
- 발령/인사이동 이력 관리
- 분기별/연간 평가 기록
- 교육 이력 관리
- 근태 정보 관리

**알림 시스템**
- 계약 요청/승인/거절 알림
- 데이터 동기화 완료 알림
- 직원 정보 변경 알림

**감사 로그**
- 모든 중요 작업 이력 추적
- 직원 정보 접근 로그
- 계약 변경 이력
- 데이터 동기화 로그

#### 3.1.2 개인 계정 기능

**통합 프로필 관리**
- 개인 기본정보 입력 및 관리
- 학력, 경력, 자격증, 어학, 병역 정보
- 포트폴리오 및 자기소개서 작성 (AI 글 다듬기 지원)
- 사진 업로드 (최대 5장, 대표 사진 선택)

**계약 관리**
- 여러 법인과 계약 체결 가능
- 계약 요청 수락/거절
- 계약별 데이터 공유 범위 설정
- 계약 이력 관리

**회사 인사카드 조회**
- 계약된 법인의 인사카드 확인
- 급여 정보, 복리후생 정보 조회
- 근무 이력 확인

**데이터 공유 통제**
- 법인별 공유 항목 선택 (기본정보, 학력, 경력, 자격증 등)
- 실시간 동기화 활성화/비활성화
- 데이터 공유 로그 확인

**개인정보 공개 설정**
- 이메일, 전화번호 공개 여부
- 주소, 생년월일 공개 여부
- 사진 공개 여부

#### 3.1.3 직원 서브 계정 기능

**법인 생성 계정**
- 법인 관리자가 생성한 계정으로 시작
- 기본 정보 입력 후 계약 체결 필수
- 법인 인사 시스템 내 프로필 관리

**개인 계정 전환 (향후 기능)**
- 직원 서브 계정을 개인 계정으로 전환
- 퇴사 후에도 자신의 데이터 소유 유지
- 전환 후 기존 법인과 계약 관계 유지

### 3.2 차별화 요소

#### 3.2.1 개인-법인 계약 시스템

**양방향 플랫폼 구조**
- 개인과 법인이 동등한 주체로 참여
- 계약 기반 투명한 데이터 공유
- 개인이 자신의 데이터 소유권 보유

**데이터 공유 설정**
- 계약별 세밀한 공유 범위 설정
- 기본정보, 학력, 경력, 자격증 등 항목별 제어
- 실시간 동기화 옵션

**계약 생애주기 관리**
- 요청 → 승인 → 활성 → 종료 전 과정 관리
- 계약 종료 후 데이터 보관 정책
- 데이터 동기화 로그 추적

#### 3.2.2 AI 문서 처리

**OCR 기반 자동 입력**
- Google Cloud Vision API 활용
- 이력서, 자격증, 명함 자동 인식
- 수기 입력 시간 80% 단축

**문서 분석 및 추출**
- Google Cloud Document AI 활용
- 근로계약서, 급여명세서 자동 분석
- 구조화된 데이터 자동 추출

**AI 텍스트 생성**
- Google Gemini API 활용
- 자기소개서 글 다듬기
- 업무 기술서 자동 생성

**로컬 AI 연동 (선택적)**
- 로컬 LLM 서버 연동 가능
- 기업 내부 데이터 보안 강화
- 비용 절감 옵션 제공

#### 3.2.3 멀티테넌시 SaaS 모델

**법인별 독립 데이터 관리**
- PostgreSQL 기반 테넌트 격리
- Company 단위 데이터 분리
- 조직도 기반 계층적 접근 제어

**확장 가능한 아키텍처**
- Flask 기반 3-Tier 레이어 아키텍처
- Repository Pattern + Service Layer
- RESTful API 설계

**보안 및 권한 관리**
- 역할 기반 접근 제어 (admin, manager, employee)
- 계정 타입별 권한 분리 (corporate, personal, employee_sub)
- 데코레이터 기반 권한 검증

### 3.3 기술적 우위

**Backend**
- Flask 2.3.0+ 기반 경량 프레임워크
- SQLAlchemy ORM으로 데이터베이스 추상화
- Alembic 마이그레이션으로 스키마 버전 관리

**Frontend**
- Jinja2 서버 사이드 렌더링
- Vanilla JavaScript ES6+ 모듈화
- 재사용 가능한 컴포넌트 구조

**Database**
- PostgreSQL 관계형 데이터베이스
- 33개 테이블, 명확한 관계 설정
- 인덱스 최적화 및 쿼리 성능 관리

**Infrastructure**
- Docker 컨테이너 배포 준비
- Nginx 리버스 프록시
- Gunicorn WSGI 서버

**AI Services**
- Google Gemini API (텍스트 생성)
- Google Cloud Vision API (OCR)
- Google Cloud Document AI (문서 분석)
- 로컬 LLM 연동 옵션

### 5.2 아키텍처

#### 레이어 구조

```
┌─────────────────────────────────┐
│   Presentation Layer             │
│  (Flask Blueprints + Jinja2)    │
└─────────────┬───────────────────┘
              ↓
┌─────────────────────────────────┐
│   Business Logic Layer           │
│  (Service Layer)                 │
└─────────────┬───────────────────┘
              ↓
┌─────────────────────────────────┐
│   Data Access Layer              │
│  (Repository Pattern)            │
└─────────────┬───────────────────┘
              ↓
┌─────────────────────────────────┐
│   Database Layer                 │
│  (PostgreSQL)                    │
└─────────────────────────────────┘
```

#### 주요 디자인 패턴

- **Factory Pattern**: App Factory로 환경별 설정 분리
- **Repository Pattern**: 데이터 접근 로직 캡슐화
- **Service Layer Pattern**: 비즈니스 로직 중앙화
- **Decorator Pattern**: 인증/권한 검증
- **Adapter Pattern**: 개인/법인 프로필 통합
