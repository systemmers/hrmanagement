## 1. 분석 개요

### 1.1 원본 문서 구조 문제점
- 중복 내용이 다수 존재 (예: 관리자 테이블 기능 반복 서술)
- 체계적 분류 없이 자유 형식으로 작성
- 구현 상태 표시 없음
- 우선순위 미지정
- hrm_checklist.md와 dev_checklist.md 내용 중복

### 1.2 분석 방법
- 현재 프로젝트 코드베이스와 체크리스트 항목 대조

- 구현 완료/진행중/미구현 상태 분류

```
섹션 1: 개인 기본정보 (_personal_info.html)
섹션 2: 소속정보 (_organization_info.html)
섹션 3: 계약정보 (_contract_info.html)
섹션 4: 학력정보 (_education_info.html)
섹션 5: 경력정보 (_career_info.html)
섹션 6: 자격증/면허 (_certificate_info.html)
섹션 7: 언어능력 (_language_info.html)
섹션 8: 병역정보 (_military_info.html)
섹션 9: 가족사항 (_family_info.html)
섹션 10: 급여정보 (_salary_info.html)
섹션 11: 4대보험 (_insurance_info.html)
섹션 12: 프로젝트 참여 (_project_info.html)
```

#### 체크리스트 요구사항 vs 현재 상태

| 요구사항 | 현재 상태 | 비고 |
|----------|----------|------|
| 사진 5장 등록/선택 | 부분구현 | 단일 사진만 지원 |
| 명함 앞뒤 업로드 | 구현완료 | API 존재 |
| 주민번호 -> 생년월일/성별 | 미구현 | JavaScript 로직 필요 |
| 사번 자동부여 | 미구현 | 설정 기반 패턴 생성 |
| 부서 트리선택 -> 자동입력 | 부분구현 | 선택 후 필드 연동 미완 |
| 계좌정보 기본정보 포함 | 미구현 | 필드 추가 필요 |
| 영어이름 필드 | 구현완료 | english_name |
| 퇴사일 필드 제거 | 미확인 | 검토 필요 |

## 5. 우선순위별 개발 로드맵

### Phase 1: 핵심 업무 기능 (HIGH)
1. **근로계약서 템플릿 시스템**
   - 템플릿 CRUD
   - 직원정보 자동 매핑
   - PDF 생성

2. **급여 계산 완성**
   - 포괄임금제 계산기 로직
   - 통상임금 자동계산
   - 급여명세서 출력

3. **엑셀 일괄 처리**
   - 급여 데이터 업로드
   - 근태 데이터 업로드
   - 검증/검산 기능

### Phase 2: 입력 편의 기능 (MEDIUM)
4. **자동화 기능**
   - 주민번호 파싱 (생년월일/성별)
   - 사번 자동생성
   - 조직선택 -> 필드 자동입력

5. **주소 검색**
   - Daum 주소 API 연동

6. **사진 관리 개선**
   - 다중 사진 등록
   - 대표 사진 선택

### Phase 3: 확장 기능 (LOW)
7. **테이블 커스터마이징**
   - 칼럼 순서/노출 설정
   - 프리셋 저장/로드

8. **명함 자동생성**
   - 템플릿 기반 이미지 생성

9. **커스텀 필드**
   - 사용자 정의 필드 추가 기능
