정보 필드를 모두 채워 오류확인
- 개인, 법인 직원의 db의 필드를 모두 채워라.
- 계약이 성립되면 법인 직원리스트에 노출되어야 하고, 개인도 회사 인사카드를 볼수 있어야 한다. 법인 직원 구조를 가져옴
- 법인 직원 등록시 계정등록과 직원등록이 함께 이뤄질 수 있도록 하며, 기본 정보섹션위에 첫번쨰 섹션으로 계정정보를 추가한다.
- 계정기본 정보는 모두같은 필드를 가지며 이외 소속정보를 분리한다. 예를들어 내선, 회사이메일등 소속에 관련된것 > 검토
- 혈액형과 마찬가지로 형식으로 조건으로 걸어둔 필드정보도 모두 표기한다. > 비교 확인후 선벼적으로 조건으로 처리예정
- 모든 필드 및 필드 라별도 비교하여 검토

사진 및 명함 업로드 기능 체크
- 무조건 기본 개인 기본정보에는 명함은 제외
- 사진은 업로드 기능 비교, 5장 까지 업로드 가능하고 메인 사진으로 선택할 수 있음

1. 회사 정보 관리
회사 기본정보 (상호, 사업자등록번호, 대표자, 주소, 연락처)
회사 로고 및 이미지
사업장 정보 (다중 사업장 관리)
2. 조직 구조 관리
조직도 설정 (company/division/department/team/unit)
직급 체계 관리
직위 관리
직책 관리
직무 및 업무분장
3. 직원 계정 관리
직원 계정 생성/정지/삭제
권한 관리 (admin/manager/employee)
계정 상태 관리
하위 계정 수 제한 설정
4. 문서 템플릿 관리
근로계약서 템플릿
급여명세서 템플릿
전자문서 템플릿 (해고장, 통지서 등)
명함 템플릿 (직원 명함 공유 설정)
5. 분류 옵션 관리
계약 유형 (정규직, 계약직, 인턴 등)
근무 형태 (전일제, 시간제 등)
부서/팀 카테고리
기타 드롭다운 옵션
6. AI 서비스 설정
AI 서비스 선택 (일반 상용/공급자 AI/로컬 설치)
OCR 문서 분석 설정
AI 글 다듬기 기능 활성화
7. 파일 관리 설정
파일 저장 방식
파일 네이밍 규칙
첨부서류 암호화 설정
파일 용량 제한
8. 계약 관리 설정
전자계약서 설정
계약 승인 프로세스
계약 알림 설정
9. 로그 및 보안
로그 기록 설정
접근 권한 로그
데이터 백업 설정
소프트삭제/하드삭제 정책
10. 시스템 설정
이메일 도메인 설정 (법인 계정 인증용)
메시지 발송 설정
알림 설정
언어 및 지역 설정
이유: 현재 문서에서 언급된 "조직 및 직위, 직급, 직책", "문서 템플릿", "AI 서비스 선택", "첨부서류 암호화" 등의 요구사항과 코드베이스의 조직 구조(organization.py), 계약 관리(contract.py), 권한 관리(decorators.py) 등을 반영했습니다.
