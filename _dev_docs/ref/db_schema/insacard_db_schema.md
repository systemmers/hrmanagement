## 5. 데이터 관계 및 제약사항

### 5.1 필수 입력 항목
**기본정보**:
- 성명(한글), 주민등록번호, 생년월일, 휴대전화, 입사일, 소속, 부서, 직급, 근무형태, 계약형태

**이력정보**:
- 최종학력 (학력구분, 학교명, 졸업년월, 졸업유무)

**인사기록**:
- 근로계약 이력 (계약일, 계약구분, 계약기간)
- 연봉계약 이력 (계약일, 계약연도, 연봉)

### 5.2 자동 계산 항목
- 나이 (생년월일 기준)
- 재직기간 (입사일 기준)
- 경력기간 (입사일 ~ 퇴사일)
- 참여기간 (시작일 ~ 종료일)
- 총 급여 (기본급 + 각종 수당)
- 연차 잔여일수 (발생일수 - 사용일수)
- 근무시간 (출근시간 ~ 퇴근시간)

### 5.3 데이터 연관관계
- 직원 기본정보 ← 모든 이력 및 기록 정보의 기준
- 소속정보 → 조직구조 (부서코드 참조)
- 직급 → 직급체계 (직급코드 참조)
- 계약정보 → 근로계약 이력 (최신 계약 반영)
- 연봉정보 → 연봉계약 이력 (최신 계약 반영)
- 인사이동 → 소속정보 변경 이력
- 프로젝트 참여 이력 ↔ 유사사업 참여경력 (재직 중 프로젝트)

### 5.4 유효성 검증
- 주민등록번호: 13자리 숫자 형식 및 유효성 검사
- 이메일: 이메일 형식 검증
- 전화번호: 전화번호 형식 검증
- 날짜: 논리적 순서 검증 (시작일 ≤ 종료일)
- 비율/백분율: 0~100 범위 검증
- 금액: 양수 검증

## 참고사항

1. 본 문서는 인사카드 시스템의 데이터 구조를 정의한 것으로, 실제 데이터베이스 테이블 설계 시 정규화 및 성능 최적화를 고려하여 구현해야 합니다.

2. 드롭다운 상수값은 시스템 코드 테이블로 관리하여 유지보수성을 높이는 것을 권장합니다.

3. 자동계산 항목은 뷰(View) 또는 계산 필드로 구현하여 데이터 정합성을 유지합니다.

4. 개인정보 보호법에 따라 민감정보(주민등록번호, 급여정보 등)는 암호화하여 저장해야 합니다.

5. 이력 정보는 시간순으로 정렬하여 조회할 수 있도록 타임스탬프를 포함해야 합니다.

6. 파일 첨부가 필요한 항목은 파일 저장 경로 또는 파일 ID를 저장하는 필드를 추가합니다.

7. 모든 기록 정보는 생성일시, 생성자, 수정일시, 수정자 정보를 포함하여 감사 추적이 가능하도록 합니다.
